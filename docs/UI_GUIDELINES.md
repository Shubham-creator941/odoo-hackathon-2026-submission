# UI Guidelines

This document ensures visual consistency across the application.

---

# General Principles

- Clean
- Professional
- Enterprise
- Minimal
- Consistent

Reference Products

- Odoo
- BambooHR
- Zoho People
- Linear
- GitHub

Avoid:

- Glassmorphism
- Neon Effects
- Oversized Gradients
- Decorative Animations

---

# Layout Pattern

Every management page follows:

Page Title

↓

Statistics (if applicable)

↓

Search

↓

Filters

↓

Primary Action

↓

Table

↓

Pagination

↓

Empty State

---

# Buttons

Primary

- Filled
- Brand Color

Secondary

- Outline

Danger

- Red

Success

- Green

All buttons use identical height and border radius.

---

# Tables

Every table contains:

- Search
- Filter
- Actions
- Pagination

Consistent column spacing.

---

# Forms

Every form includes:

- Labels
- Validation
- Loading State
- Success Message
- Error Message

---

# Modals

Used for:

- Confirmation
- Details
- Edit

Never place long forms inside modals.

---

# Empty States

Every module must have a meaningful empty state.

---

# Loading States

Use skeleton loaders or spinners.

Never leave blank pages.

---

# Status Badges

Active → Green

Inactive → Gray

Pending → Orange

Approved → Green

Rejected → Red

---

# Responsiveness

Desktop

Laptop

Tablet

Mobile

No broken layouts.

---

# Accessibility

- Keyboard Navigation
- Focus States
- Semantic HTML
- Sufficient Color Contrast
